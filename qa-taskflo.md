## BUGS

### BUG-K1: `status = 'open'` is invalid status value
**Severity: High**
**Line: 3783**

In the "Move to column" dropdown handler:
```js
if (col.id === 'completed') { task.status = 'completed'; } else { task.status = 'open'; }
```

The value `'open'` is not a valid task status. The rest of the codebase uses `'pending'` and `'in_progress'`. Tasks moved via the dropdown (to todo, christopher, claude, assistance, parking) get `status = 'open'`, which means:
- `getTaskColumn()` won't recognize them correctly
- `filterTasks()` may misplace them
- `isBlocked()` and other status checks fail silently

**Fix:** Replace with correct status values:
```js
if (col.id === 'completed') { task.status = 'completed'; }
else if (col.id === 'todo' || col.id === 'parking') { task.status = 'pending'; }
else { task.status = 'in_progress'; }
```

### BUG-K2: `saveTasksToGithub()` called without arguments in move handler
**Severity: Medium**
**Line: 3790**

```js

```

`saveTasksToGithub()` expects `(tasksData, reason)`. Called without arguments: `tasksData` is `undefined`. Inside the function, the first thing it does is `tasksData.forEach(...)` — this throws a TypeError, silently failing the save.

**Fix:** Change to `saveTasksToGithub(tasks, 'Move task');`

### BUG-K5: Task "completed_at" not cleared on reopen
**Severity: Low**
**Lines: 3819-3836**

When reopening a task (`toggleTaskComplete`):
```js

if (task.status === 'completed') {
    task.completed_at = new Date().toISOString();
} else {
    // task.completed_at is NOT cleared

```

The `completed_at` timestamp remains set even after reopening, which could confuse stats/sorting that use it.

**Fix:** Add `task.completed_at = null;` in the else branch (this is done in `handleDrop` but not in `toggleTaskComplete`).

## UX ISSUES

### UX-K1: Move-to menu opens upward and may clip off screen
The `.move-to-menu` is positioned `bottom: 100%` — it opens above the button. For tasks in the top rows of a column, this may render off-screen.
**Fix:** Add viewport detection and conditionally use `top: 100%` when near the top.

### UX-K2: No undo for destructive actions
Moving a task to Completed or deleting it has no undo. Users frequently mis-click.
**Fix:** Add a brief "Undo" toast (5 seconds) after any destructive action.

### UX-K3: Batch operation bar has no "Deselect All" button
The batch bar appears when shift-clicking tasks, but there's no obvious way to cancel the selection other than re-clicking each task.
**Fix:** Add a "✕ Deselect" or "Cancel" button to the batch bar.

### UX-K4: "Move to column" button shows `➤` (arrow character) with no label
The move button (`\u27A4`) has no text — just an arrow character. First-time users won't know what it does.
**Fix:** Either add a tooltip that's more visible on hover, or change the button to show "Move ▾".

### UX-K5: Empty columns show "No tasks match this filter" even when no filter is active
When a column genuinely has no tasks and no filter is applied, the message reads "No tasks match this filter" — implying the user did something wrong.
**Fix:** Show "No tasks here yet" when no filter is active, "No tasks match this filter" only when filters are applied.

### UX-K6: Priority filter only filters by exact match, not "at least"
Selecting "High" priority shows only priority 1 tasks — it doesn't include priority 1 tasks in mixed views. Considering "High" normally means "show me urgent things", users expect Priority 1+ or at minimum a visible explanation.
**Fix:** Consider "show at least this priority" behavior or clarify the filter label.

### UX-K7: No visual feedback when GitHub save is in progress
There's a sync indicator for offline state but no spinner/animation while an active save is happening. Users don't know if their change is being saved.
**Fix:** Show a subtle pulsing "Saving..." indicator during `_isSaving = true`.

### UX-K8: Clicking assignee label to cycle owners is a hidden interaction
Tapping the assignee chip to cycle Christopher → Claude → Assistance is a non-obvious gesture. No tooltip or hint exists.
**Fix:** Add a tooltip "Click to reassign" or replace with a dropdown.
